**3. Опис кожного реалізованого алгоритму:**
### Selection Sort (Сортування вибором)
* **Як виконується:** Алгоритм візуально ділить масив на відсортовану і невідсортовану частини. Він проходить по невідсортованій частині, знаходить там найменший елемент і міняє його місцями з першим елементом невідсортованої частини.
* **Ключові етапи:** 1) Пошук мінімального елемента.
    2) Обмін (Swap) знайденого мінімуму з поточним елементом.
    3) Зсув межі відсортованої частини на один крок вправо.
* **Складність:** Найкращий випадок: $O(n^2)$. Найгірший випадок: $O(n^2)$.

### Quick Sort (Швидке сортування)
* **Як виконується:** Використовує підхід "розділяй і володарюй". Обирається один елемент як "опорний" (pivot). Усі елементи, менші за нього, переносяться ліворуч, а більші — праворуч. Потім алгоритм рекурсивно викликається для лівої та правої частин.
* **Ключові етапи:** 1) Вибір опорного елемента (pivot).
    2) Розбиття масиву (Partition) на дві підмножини відносно півота.
    3) Рекурсивне сортування лівої та правої підмножин.
* **Складність:** Найкращий випадок: $O(n \log n)$. Найгірший випадок: $O(n^2)$ (виникає, якщо масив уже відсортований, а півотом завжди обирається крайній елемент).

### Merge Sort (Сортування злиттям)
* **Як виконується:** Також "розділяй і володарюй", але працює інакше: він рекурсивно ділить масив строго навпіл, поки не залишаться шматочки по одному елементу (які вважаються відсортованими). Потім він поступово зливає ці шматочки у більші відсортовані масиви за допомогою додаткової пам'яті.
* **Ключові етапи:** 1) Рекурсивний поділ масиву навпіл.
    2) Злиття (Merge) двох сусідніх відсортованих підмасивів в один (потребує буферного масиву).
* **Складність:** Найкращий випадок: $O(n \log n)$. Найгірший випадок: $O(n \log n)$.

### Hybrid Sort (Гібридне сортування)
* **Як виконується:** Це комбінація рекурсивного алгоритму (наприклад, Merge Sort) та простого (наприклад, Selection Sort). Гібрид використовує рекурсію для швидкого розбиття великих масивів, але щойно розмір підмасиву стає меншим за певне порогове значення (Threshold), він перемикається на простий алгоритм для економії ресурсів на викликах функцій.
* **Ключові етапи:** 1) Перевірка розміру поточного підмасиву.
    2) Якщо розмір більший за поріг — рекурсивний виклик.
    3) Якщо менший — базове сортування.
* **Складність:** Залежить від базових алгоритмів. В нашому випадку — Найкращий: $O(n \log n)$, Найгірший: $O(n \log n)$ або $O(n^2)$ (залежно від того, чи базується на Merge, чи на Quick).

### Counting Sort (Сортування підрахунком)
* **Як виконується:** Не порівнює елементи між собою. Замість цього він створює додатковий масив-"блокнот", де підраховує, скільки разів зустрічається кожне значення. Потім він перетворює ці кількості на індекси позицій і розставляє елементи на їхні місця.
* **Ключові етапи:** 1) Знаходження максимального значення (для розміру блокнота).
    2) Підрахунок частоти входжень кожного ключа.
    3) Обчислення префіксних сум (для збереження стабільності).
    4) Прохід по вихідному масиву з кінця в початок і розставлення елементів.
* **Складність:** Найкращий випадок: $O(n + k)$. Найгірший випадок: $O(n + k)$ (де $n$ — кількість елементів, а $k$ — діапазон можливих значень).

### Radix Sort (Порозрядне сортування)
* **Як виконується:** Використовує Counting Sort як допоміжний інструмент для сортування великих чисел. Він сортує масив по одній цифрі за раз: спочатку за одиницями, потім за десятками, сотнями і так далі до найстаршого розряду. Завдяки стабільності Counting Sort порядок не руйнується.
* **Ключові етапи:** 1) Знаходження найбільшого числа (для визначення кількості розрядів).
    2) Цикл `for`, який проходить по кожному розряду (множники 1, 10, 100...).
    3) Виклик стабільного міні-Counting Sort для поточної цифри на кожній ітерації.
* **Складність:** Найкращий випадок: $O(d \cdot (n + k))$. Найгірший випадок: $O(d \cdot (n + k))$ (де $d$ — кількість розрядів у найбільшому числі).

**4. Приклади використання алгоритмів:**
* **Selection Sort:**
    1) Програмування мікроконтролерів або вбудованих систем (embedded systems) з дуже обмеженим ресурсом флеш-пам'яті, де операції запису (обміну) коштують "дорого" і їх треба мінімізувати.
    2) Використовується в навчальних цілях для ілюстрації базових концепцій алгоритміки.

* **Quick Sort:**
    1) Вбудовані функції сортування в стандартних бібліотеках мов програмування (наприклад, класична функція `qsort()` в мові C).
    2) Робота з величезними масивами даних в оперативній пам'яті (наприклад, при обробці телеметрії чи лог-файлів), оскільки він дуже дружній до кешу процесора.

* **Merge Sort:**
    1) "Зовнішнє сортування" (External sorting) у базах даних, коли масив інформації (наприклад, таблиця на 100 ГБ) не влазить в оперативну пам'ять, і його доводиться читати та зливати частинами прямо з жорсткого диска.
    2) Сортування зв'язних списків (Linked Lists), оскільки для нього не потрібен довільний доступ до елементів пам'яті.

* **Hybrid Sort:**
    1) Рушій сортування `std::sort` у сучасному C++ (використовує Introsort: суміш Quick, Heap та Insertion sort).
    2) Стандартне сортування у мовах Python та Java (використовують Timsort: суміш Merge та Insertion sort), яке оптимізовано під масиви даних з реального світу.

* **Counting Sort:**
    1) Алгоритми комп'ютерної графіки та обробки зображень для сортування пікселів за кольором (де діапазон значень RGB завжди чітко обмежений від 0 до 255).
    2) Сортування записів у базах даних за невеликими характеристиками (наприклад, сортування тисяч користувачів за віком від 0 до 120 років).

* **Radix Sort:**
    1) Високошвидкісне мережеве обладнання (маршрутизатори), де потрібно миттєво сортувати і фільтрувати величезні масиви IP-адрес.
    2) Біоінформатика: при побудові суфіксних масивів для надшвидкого пошуку патернів та збігів у ланцюгах ДНК.
